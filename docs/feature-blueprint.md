# AI SRE Investigator — 完整功能蓝图 (Feature Blueprint)

> 阶段：**功能发现与路线图规划**
> 输入：PRD V1.0 MVP + brainstorm-ideas.md（15 创意 Top 5）+ tech-design + spike 结论
> 目标：从 PM / Designer / Engineer 三视角做全面功能拆解，按阶段分层（MVP → V1.5 → V2 → V3），每个功能标注 UI 影响，直接喂给原型设计。
> 与 brainstorm-ideas.md 的关系：brainstorm 是"点子发散"，本文档是"点子落地成功能矩阵 + UI 页面映射"。

---

## 0. 阶段定义

| 阶段 | 定位 | 目标用户 | 信任门槛 |
|------|------|----------|----------|
| **MVP** | 核心闭环验证：大白话 → QL → 证据 → RCA | 内部灰度 5-10 人 | 能跑通、不崩 |
| **V1.5** | 体验与信任加固：让开发敢独立用 | 扩大到 50 人 | 敢据此行动 |
| **V2** | 组织级沉淀：越用越聪明 | 全公司推广 | 被组织采纳 |
| **V3** | 平台化与生态：MCP / 开放扩展 | 跨团队 / 开源 | 成基础设施 |

---

## 1. MVP（核心闭环）

> 目标：证明"大白话 → 准确 QL → 有用的 RCA"这条链路成立。
> Spike 已验证：PromQL 准确率 93%，语法 100% 通过，方向可行。

### MVP-F1 自然语言故障申告（Chat 入口）

| 维度 | 内容 |
|------|------|
| 来源 | PRD §3.1 |
| 功能 | 用户用一句话描述故障（如"payment-service 刚才为什么大量 500"），Agent 接收并启动排查 |
| UI 页面 | **Chat 页** — 主交互界面，类似 ChatGPT 的输入框 + 对话流 |
| UI 组件 | 输入框（支持多行 / 快捷模板）、提交按钮、当前会话状态指示器（思考中 / 查询中 / 生成报告中） |
| 依赖 | 无（入口功能） |

### MVP-F2 自动 QL 转化与执行（Agent 核心）

| 维度 | 内容 |
|------|------|
| 来源 | PRD §3.1 + §5（Tool Specs） |
| 功能 | Agent 将自然语言转为 PromQL / LogQL / TraceQL，调用 Mimir / Loki / Tempo 三探针执行 |
| 后端 | `agent/loop.py`（Tool-Calling 循环）+ `tools/mimir.py` / `loki.py` / `tempo.py` |
| UI 页面 | **Timeline 面板**（Chat 页右侧或下方）— 展示 Agent 每步调用的思考与查询 |
| UI 组件 | 步骤卡片列表（调用哪个探针、生成什么 QL、返回什么结果）、每步状态灯（成功 / 失败 / 自修正中） |
| 依赖 | MVP-F1 |

### MVP-F3 异步推理进度流（Timeline）

| 维度 | 内容 |
|------|------|
| 来源 | PRD §3.1（前端进度流） |
| 功能 | SSE 流式推送 Agent 多步推理过程，前端实时渲染每一步 |
| 后端 | SSE endpoint (`GET /api/session/{id}/stream`) |
| UI 页面 | **Timeline 面板** |
| UI 组件 | 时间轴（垂直）、每步卡片（探针名 + QL 代码块 + 结果摘要 + 耗时）、加载动画、失败标红 |
| 依赖 | MVP-F2 |
| 与 brainstorm 关系 | D-1 证据可下钻 Timeline 的 MVP 基础版 |

### MVP-F4 结构化 RCA 报告生成

| 维度 | 内容 |
|------|------|
| 来源 | PRD §3.1 |
| 功能 | Agent 排查完成后，聚合指标 / 日志 / 链路证据，生成 Markdown 格式 RCA 报告 |
| UI 页面 | **RCA 报告页** — 独立全屏视图或 Chat 页内嵌面板 |
| UI 组件 | Markdown 渲染器（React Markdown）、报告结构导航（服务与问题 / 时间线 / 证据 / 根因与建议）、证据锚点跳转 |
| 依赖 | MVP-F3 |

### MVP-F5 SafeToolExecutor 四层包装

| 维度 | 内容 |
|------|------|
| 来源 | ADR-004 + tech-design-ql-selfheal.md |
| 功能 | 预算预检 + 查询缓存 + QL 预校验 + 失败自修正，对所有工具透明生效 |
| 后端 | `agent/safe_executor.py` |
| UI 页面 | Timeline 面板中体现自修正过程 |
| UI 组件 | 自修正子步骤（"查询失败 → 自动修正 → 重试成功"折叠展开）、预算进度条（剩余 token / 查询次数） |
| 依赖 | MVP-F2 |
| 与 brainstorm 关系 | E-1 + E-2 落地 |

### MVP-F6 会话管理

| 维度 | 内容 |
|------|------|
| 来源 | PRD §6（api/chat.py 路由） |
| 功能 | 创建会话、轮询状态、历史会话列表 |
| 后端 | `POST /api/chat`（创建）+ `GET /api/session/{id}`（状态） |
| UI 页面 | **会话列表侧边栏**（类 ChatGPT 左侧栏） |
| UI 组件 | 会话列表（标题 = 用户申告的前 30 字 + 时间戳）、新建会话按钮、会话状态标记（进行中 / 已完成 / 部分完成） |
| 依赖 | MVP-F1 |

## 2. V1.5（信任与体验加固）

> 目标：让开发敢据此行动，而不是"看一眼还是去问 SRE"。
> 核心风险：信任。透明可验证是采纳的命门。

### V15-F1 证据可下钻 Timeline（升级版）

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm D-1 |
| 功能 | Timeline 每一步从"文字摘要"升级为可展开的证据卡：指标 mini 图表（标注波峰）、日志样本（高亮异常行）、Trace 瀑布图（标红慢 Span） |
| UI 页面 | **Timeline 面板**（升级） |
| UI 组件 | 指标 sparkline 图（内嵌 Mimir 数据可视化）、日志查看器（语法高亮 + 异常行标红）、Trace 瀑布图（Span 时间轴 + 耗时色阶）、证据展开 / 折叠 |
| 依赖 | MVP-F3 |
| UI 原型重点 | 这是 MVP → V1.5 最大的视觉跃升，原型需重点设计三种证据卡片的展示形态 |

### V15-F2 对话式追问（Conversational Follow-up）

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm D-2 |
| 功能 | 围绕 RCA 继续自然语言下钻："把 p99 拉长 3 小时"、"展开这个 TraceID 的下游"，Agent 复用上下文与缓存 |
| UI 页面 | **Chat 页**（延续对话） |
| UI 组件 | 追问快捷按钮（基于当前 RCA 上下文动态生成，如"查看更多日志"、"拉长时间窗口"）、追问历史折叠 |
| 依赖 | MVP-F4, V15-F1 |

### V15-F3 置信度标注与不确定性表达

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm D-5 + PM-2 |
| 功能 | RCA 每个结论标注置信度（高 / 中 / 低），证据不足时标注"建议进一步查 X" |
| UI 页面 | **RCA 报告页**（增强） |
| UI 组件 | 置信度色阶标签（绿 / 黄 / 红 badge）、结论旁的"证据强度"指示器、建议下一步操作的可点击链接 |
| 依赖 | MVP-F4 |

### V15-F4 优雅收尾与部分 RCA

| 维度 | 内容 |
|------|------|
| 来源 | ADR-005 |
| 功能 | 预算耗尽或部分失败时，用已有证据生成"部分 RCA"，顶部标注证据缺口 |
| UI 页面 | **RCA 报告页** |
| UI 组件 | 顶部警告横幅（"本报告因时间限制未能完成以下查询"）、缺失查询列表、已获取证据 vs 缺失证据的视觉区分 |
| 依赖 | MVP-F5 |

### V15-F5 模拟故障上手引导

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm D-3 |
| 功能 | 新用户首次进入，用一个预置模拟故障（如 demo 服务的 OOM）2 分钟跑通完整流程 |
| UI 页面 | **引导页 / 首次登录页** |
| UI 组件 | 模拟故障卡片列表（选一个试排查）、引导提示气泡（指向输入框 / Timeline / 报告）、"跳过引导"按钮 |
| 依赖 | MVP 全部功能就绪 |

### V15-F6 一键分享 RCA

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm D-4 |
| 功能 | RCA 生成可分享链接（含证据快照，数据脱敏），支持发 Slack / 钉钉 |
| UI 页面 | **RCA 报告页**（增强） |
| UI 组件 | 分享按钮（生成 permalink）、频道快捷发送按钮（Slack / 钉钉 icon）、分享链接 QR 码 |
| 依赖 | MVP-F4 |

## 3. V2（组织级沉淀与越用越聪明）

> 目标：把少数 SRE 的肌肉记忆变成全员可调用的组织资产。
> 核心命题：数据飞轮转起来。

### V2-F1 RCA 知识库与模式记忆（Incident Memory）

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm PM-1（Top 1 创意） |
| 功能 | 每次成功排查的 RCA 结构化入库（症状 → 查询路径 → 根因），形成组织级故障图谱。下次相似症状命中时复用历史"黄金排查路径" |
| UI 页面 | **知识库页** — 全新页面 |
| UI 组件 | 故障图谱可视化（症状 → 根因的网络图）、历史 RCA 列表（搜索 + 筛选 + 标签）、"命中历史模式"提示卡（排查时出现"上次类似故障是 X"）、RCA 收藏 / 标注 |
| 依赖 | MVP-F4（需要结构化 RCA 数据） |
| 验证假设 | H1: 真实故障"症状→根因"模式可被结构化复用；H2: 开发愿意花 30 秒确认根因（数据飞轮输入） |

### V2-F2 黄金路径排查剧本库

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm PM-3（Top 3 创意） |
| 功能 | 为高频故障（OOM / 连接池打满 / 依赖超时 / GC 停顿 / 限流）预置剧本式排查流：触发条件 → 标准查询 → 判定证据 → 常见根因 |
| UI 页面 | **剧本库管理页**（SRE 用） + 排查时自动套用提示 |
| UI 组件 | 剧本列表（触发条件 + 覆盖率统计）、剧本详情编辑器（步骤式）、排查时"命中剧本"通知（"检测到疑似 OOM，已套用标准排查路径"）、命中率仪表盘 |
| 依赖 | V2-F1（知识库数据驱动剧本生成） |
| 验证假设 | H1: Top 10 故障模式覆盖 ≥60% 事件 |

### V2-F3 服务画像与归属模型

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm PM-4 |
| 功能 | 采集服务元数据（owner team / 依赖拓扑 / SLO / 历史故障率），RCA 自动定位责任方并打标 |
| UI 页面 | **服务目录页** + RCA 报告中自动填充归属信息 |
| UI 组件 | 服务依赖拓扑图（节点 = 服务，边 = 调用关系）、服务详情卡（owner / SLO / 近 7 天故障率）、RCA 中的"责任方"自动打标 badge |
| 依赖 | MVP-F2（需要知道查了哪些服务） |
| MVP 变体 | V1 阶段先用静态配置（YAML 服务名映射），V2 接 CMDB |

### V2-F4 投资回报仪表盘

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm PM-5 |
| 功能 | 量化产品自身价值：自闭环率、平均排查时长、SRE 被打扰次数下降、Token / 查询成本 |
| UI 页面 | **价值仪表盘页**（管理员 / SRE 负责人用） |
| UI 组件 | KPI 卡片（自闭环率 / MTTR 改善 / 成本 / 采纳率）、趋势折线图（周 / 月）、价值简报导出按钮 |
| 依赖 | MVP-F6（需要会话数据）+ V2-F3（服务画像） |
| 与 brainstorm 关系 | 对应 discovery-gaps #3 痛点基线 |

### V2-F5 置信度感知的人机协同升级

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm PM-2（Top 5 创意） |
| 功能 | 低置信度或证据缺失时，自动生成"交接卡"一键转给 on-call SRE，附证据链 |
| UI 页面 | **交接卡弹窗 / 页面** |

| 后端 | 会话级缓存（跨 Tool 调用结果复用）+ 上下文窗口管理 |
| UI 页面 | **Chat 页**（增强） |
| UI 组件 | 对话上下文面包屑（当前排查的服务 / 时间范围 / 已查探针）、上下文切换器（多会话并行） |
| 依赖 | V15-F2 |

## 4. V3（平台化与生态扩展）

> 目标：从"一个排查工具"演进为"可扩展的 SRE AI 平台"。
> 核心方向：MCP 协议迁移、开放工具生态、多模型支持。

### V3-F1 MCP 协议迁移

| 维度 | 内容 |
|------|------|
| 来源 | ADR-001 + ADR-003 |
| 功能 | 从原生 Tool-Calling 平滑迁移到 MCP (Model Context Protocol)，tool 变成 MCP server |
| 后端 | ToolSpec transport 层替换（业务逻辑零改动） |
| UI 影响 | 无直接 UI（纯架构演进），但间接提升性能和可扩展性 |
| 依赖 | ADR-003 ToolSpec 契约已为此铺路 |

### V3-F2 可插拔工具注册表

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm E-3 |
| 功能 | 新增工具（Prometheus / Elasticsearch / Kubernetes / 云厂商 API）只需注册 ToolSpec，无需改 Agent 核心 |
| UI 页面 | **工具市场页**（管理可用工具） |
| UI 组件 | 工具列表（名称 / 状态 / 版本）、工具详情卡（支持什么查询）、启用 / 禁用开关、新增工具向导 |
| 依赖 | V3-F1 |

### V3-F3 多模型路由与对比

| 维度 | 内容 |
|------|------|
| 功能 | 支持多个 LLM 后端（云端 / 本地），按故障类型 / 成本 / 延迟路由，或同时跑两个对比取优 |
| UI 页面 | **设置页**（模型配置）+ RCA 报告中标注使用了哪个模型 |
| UI 组件 | 模型选择器（下拉 / segmented control）、模型对比视图（A/B 结果并排）、成本 / 准确率对比表 |
| 依赖 | 无硬依赖 |

### V3-F4 流式可恢复会话

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm E-4 |
| 功能 | 会话状态落盘，支持断线重连、长任务后台续跑、用户离开后回来继续看进度 |
| UI 页面 | **会话恢复提示** |
| UI 组件 | 重连后"恢复进度"横幅、后台任务通知、会话快照预览 |
| 依赖 | MVP-F6 |

### V3-F5 自观测与 Dogfooding

| 维度 | 内容 |
|------|------|
| 来源 | brainstorm E-5 |
| 功能 | 给 Agent 自身埋点（token / 延迟 / 成功率 / 缓存命中率），用自家 Mimir / Loki 看自己 |
| UI 页面 | **自观测仪表盘页**（开发 / SRE 团队用） |
| UI 组件 | 实时指标面板（QL 首次错误率 / 自修正成功率 / 缓存命中率 / 平均排查耗时）、告警规则配置 |
| 依赖 | MVP-F5（埋点数据） |

### V3-F6 半自动处置（建议 + 一键执行）

| 维度 | 内容 |
|------|------|
| 来源 | ADR-002 演进口 |
| 功能 | RCA 给出修复建议后，经人确认可一键执行（扩缩容 / 重启 / 改配置），需审计链路 |
| UI 页面 | **处置确认弹窗** |
| UI 组件 | 建议操作卡（操作描述 + 影响范围 + 回滚方案）、人确认 checkbox、执行进度、审计日志 |
| 依赖 | V2-F5（置信度体系成熟后才开放写权限） |
| 前提 | ADR-002 要求：只读 RCA 已建立足够信任 → 这是 V3 才考虑的演进方向 |

| **知识库页** | V2 | 故障图谱 + 历史 RCA 列表 + 模式命中提示 | P2 |
| **剧本库管理页** | V2 | 剧本列表 + 编辑器 + 命中率仪表盘 | P2 |
| **服务目录页** | V2 | 依赖拓扑图 + 服务详情卡 | P2 |
| **价值仪表盘页** | V2 | KPI 卡片 + 趋势图 | P2 |
| **交接卡** | V2 | 证据链 + 置信度 + 转交按钮 | P2 |
| **设置页** | V3 | 模型配置 + 工具管理 + 预算配置 | P3 |
| **工具市场页** | V3 | 工具列表 + 启用开关 + 新增向导 | P3 |
| **自观测仪表盘页** | V3 | 实时指标 + 告警规则 | P3 |
| **处置确认弹窗** | V3 | 建议操作 + 确认 + 审计日志 | P3 |

### 5.1 MVP 核心布局（原型第一优先级）

```text
┌─────────────┬──────────────────────────────────────────┐
│             │                                          │
│  会话列表    │            Chat 主区域                   │
│  侧边栏      │  ┌──────────────────────────────────┐    │
│             │  │  对话流（用户申告 + Agent 回复）   │    │
```

### 5.2 V1.5 增强焦点

MVP 到 V1.5 的最大视觉变化集中在 **Timeline 面板**：
- 每步从"文字卡片"变为"可展开的证据卡"
- 指标查询结果 → sparkline mini 图表 + 阈值标注
- 日志查询结果 → 日志查看器 + 异常行高亮
- 链路查询结果 → Trace 瀑布图 + 慢 Span 标红

### 5.3 设计风格方向

- **基调**：操作工具，不是消费产品。安静、密集、可扫描。
- **配色**：靛蓝瓷主题（与 PPT 一致），深色为主（SRE 偏好暗色模式）。
- **排版**：信息密集但不拥挤，优先左右分栏（侧边栏 + 主区域）。
- **交互**：SSE 实时流式更新，Timeline 步骤逐个出现。
