根因分析 置信度: 高

OrderCache 内存泄漏。 OrderCache 类使用 HashMap 缓存订单数据，但未配置任何淘汰策略（无 maxSize / TTL）。每次 putAll 调用持续往 HashMap 写入数据，而旧数据永不移除。

随着订单量增长，HashMap 持续 resize，堆内存从 400MB 线性增长至 1GiB limit。Full GC 无法回收（存活对象 ~780MB），最终触发 OutOfMemoryError。

修复建议

P0 立即将 OrderCache 从 HashMap 替换为 Caffeine，设置 maximumSize(10000) 和 expireAfterWrite(30m)
短期临时将容器内存 limit 从 1Gi 提升至 2Gi，为修复赢得窗口期
短期添加 JVM 参数 -XX:MaxHeapFreeRatio=70 -XX:MinHeapFreeRatio=40 优化 GC 行为
长期引入内存使用告警：当 Heap 使用超过 80% 持续 5min 时自动告警
