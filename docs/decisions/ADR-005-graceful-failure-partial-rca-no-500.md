# ADR-005: 优雅收尾--失败/预算耗尽生成部分 RCA，绝不抛 500

## Status
Accepted

## Date
2026-07-09

## Context
生产 SRE 排查中，Agent 不可避免会遇到：QL 自修正耗尽次数、观测后端持续 5xx、会话预算耗尽、超时。这些场景下若向前端抛 500 或静默失败，会：
- 违反 PRD 红线（绝不向前端抛 500 导致 Agent 中断）。
- 破坏组织信任--用户拿不到任何结果，下次直接回退人工。
- 可能产出"假装查完实则臆测"的错误 RCA，在生产事件中危害更大。

## Decision
**任何失败/预算耗尽场景都不抛异常、不静默失败，而是用已收集证据生成"部分 RCA"，并在报告顶部显式标注未完成的查询与证据缺口。** 具体由 SafeToolExecutor（ADR-004）的优雅收尾分支实现：

- 预算预检失败或 tool 最终 failed -> 不再发起新 tool。
- 调用 LLM 用当前已有证据生成部分 RCA。
- RCA 顶部标注：`⚠️ 本报告因预算/时间限制未能完成以下查询：[...]，结论基于部分证据。`
- 单 tool 自修正耗尽（3 次失败）-> 该证据标注缺失，RCA 如实说明而非臆测。

本决策与 brainstorm D-5（诚实不确定性 UX）一脉相承：诚实告诉用户"我没查完"，而非假装查完给错答案。

## Alternatives Considered

### 失败即抛 500 / 中断会话
- Pros：实现最简。
- Cons：直接违反 PRD 红线；用户零结果，信任崩塌；下次直接不用。
- Rejected。

### 失败即静默返回空 RCA
- Pros：不抛异常。
- Cons：用户误以为"查完没问题"，可能误导决策；不诚实。
- Rejected：SRE 场景下"看似查完实则没查"比明确报错更危险。

### 失败时强制人工转交，不产出任何报告
- Pros：安全。
- Cons：已收集的证据被浪费；SRE 仍需从零开始，未体现 Agent 的部分价值。
- Rejected：部分证据仍有价值，应保留并如实呈现，配合 brainstorm Top #5 的人工升级闸门。

## Consequences
- 正向：任何情况下用户都能拿到带证据缺口说明的结果，信任不被破坏。
- 正向：部分证据仍可辅助人工 SRE 后续排查，不浪费已完成工作。
- 正向：诚实的"未完成"标注反而强化专业可信感（与过度自信的臆测形成对比）。
- 负向：部分 RCA 可能不完整，需在 UI 上用置信度色阶（brainstorm D-5）醒目提示，避免被误当完整结论。
- 约束：LLM 在生成部分 RCA 时必须遵守"不臆测"prompt 约束，对缺失证据明确留白而非编造。
